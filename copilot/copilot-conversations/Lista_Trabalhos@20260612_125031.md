### Como usar este template:

1.  **Cria uma pasta para templates:** Se ainda não o fizeste, cria uma pasta chamada, por exemplo, `_Templates` na raiz do teu vault.
2.  **Cria o ficheiro do template:** Dentro da pasta `_Templates`, cria um novo ficheiro Markdown e cola o conteúdo acima. Podes chamá-lo de `Template - Lote.md`.
3.  **Configura o Templater:**
    *   Vai a `Settings` (Definições) do Obsidian.
    *   Clica em `Community plugins` (Plugins da comunidade) e depois em `Templater`.
    *   Em `Template folder location` (Localização da pasta de templates), seleciona a pasta `_Templates` que criaste.
4.  **Cria uma nova nota de lote:**
    *   Quando quiseres criar uma nova nota para um lote, podes usar o comando "Templater: Insert template" (Inserir template) (podes configurar um atalho de teclado para isto).
    *   Seleciona `Template - Lote.md`.
    *   O Templater irá pedir-te para preencher os campos (`tp.system.prompt`) como "Qual é a obra principal?", "Qual é o tipo de lote?", etc. Preenche-os e a nota será criada com toda a informação preenchida e a estrutura pronta.

### Explicação das funcionalidades do template:

*   **Frontmatter (topo da nota):** Contém metadados importantes que o Dataview pode usar para filtrar e organizar.

### Como Adaptar para 'x', 'P' ou 'F' nas Presenças

Atualmente, o script preenche a coluna "Presença" com `[x]` (que o Dataview interpreta como uma tarefa concluída, mas aqui é apenas um marcador visual).

Para teres mais flexibilidade (P/F/x), terás de mudar a forma como essa coluna é preenchida no script e como é interpretada.

#### Opção 1: Usar 'P' ou 'F' como texto (mais simples)

Se quiseres apenas texto, podes mudar a linha no script:

```javascript
// Linha original:
// tableRows += `| [[${file.path}|${personName}]] | [x] | 8 | 0 | [ ] | |\n`;

// Alterar para 'P' (Presente) por defeito:
tableRows += `| [[${file.path}|${personName}]] | P | 8 | 0 | [ ] | |\n`;
```

**Como funciona:**
*   Quando o ficheiro de presenças for criado, a coluna "Presença" terá um "P" para todos.
*   Depois, podes editar manualmente o ficheiro de presenças para mudar "P" para "F" (Faltou) ou "x" (se for o caso) para cada funcionário.

#### Opção 2: Usar um prompt para cada funcionário (mais interativo, mas pode ser demorado para muitos funcionários)

Se quiseres decidir a presença de cada um no momento da criação:

```javascript
// ... dentro do loop for (const file of peopleFolder.children) ...

        if (file instanceof TFile && file.extension === "md") {
            const personName = file.basename;
            // Pergunta a presença para cada funcionário
            const presenceStatus = await tp.system.prompt(`Presença para ${personName} (P/F/x)?`, "P");
            tableRows += `| [[${file.path}|${personName}]] | ${presenceStatus} | 8 | 0 | [ ] | |\n`;
        }
```

**Como funciona:**
*   Ao executar o template, depois de perguntar o nome da obra e as tarefas, ele vai perguntar a presença para **cada funcionário** da lista.
*   Isto pode ser útil se tiveres poucos funcionários, mas se tiveres muitos, pode tornar o processo de "Iniciar Dia Obra" muito longo.

#### Opção 3: Usar um checkbox `[ ]` e editar depois (o que já tens, mas com a intenção de editar)

A tua implementação atual com `[x]` na verdade é um checkbox. Se o objetivo é que o `[x]` signifique "presente" e `[ ]` signifique "ausente", então o que já tens está bom.

```javascript
// Linha atual:
tableRows += `| [[${file.path}|${personName}]] | [x] | 8 | 0 | [ ] | |\n`;
```

**Como funciona:**
*   Todos os funcionários vêm com `[x]` (presente).
*   Se alguém faltar, vais ao ficheiro de presenças e mudas o `[x]` para `[ ]` (ausente) para esse funcionário.
*   A vantagem é que o Obsidian renderiza `[x]` como um checkbox marcado e `[ ]` como um checkbox desmarcado, o que pode ser visualmente intuitivo.

### Recomendação

Para a maioria dos casos, eu recomendaria a **Opção 1 (usar 'P' por defeito e editar manualmente)** ou a **Opção 3 (usar `[x]` e editar para `[ ]`)**. A Opção 2 pode ser demasiado interativa e demorada para o início do dia.

Se optares por 'P' ou 'F', lembra-te que o Dataview não vai conseguir filtrar por "tarefas concluídas" nessa coluna, mas podes filtrar por texto (ex: `WHERE Presença = "P"`).

Escolhe a opção que melhor se adapta ao teu fluxo de trabalho!
